Привет!) В этом файле я постараюсь рассказать свою идею реализации телефонного справочника.
Сразу хочу извиниться за возможные орфографичесеские ошибки - набираю этот тест в pyCharm)

Итак. При каждом запуске приложения первое, что потребуется сделать - ввести логин и пароль.
По умолчанию - admin / admin  внижнем регистре. Нужно это для того, чтобы вносить изменения
в телефонную книгу мог не каждый желающий. Администраторы - могут и вносить новые записи в
справочник и изменять уже существующие. Пользователи, не прошедшие авторизацию - сомгут только
вносить новые.

Изначально я реализовывл json по принципу: создание словаря, где по умолчанию созданы пары
"буквы алфавита": [список {словарей}]. Типа есть 26+35 ключей, которые помогут быстрее отвечать
на вопрос, а есть ли такой пользователь в справочнике. Затем, после некоторых набросков было
принято решение забыть про эту идею, ведь храниение абонентов в алфавитном порядке поможет только
если искать пользователей мы будем только по фамилиии. Во всех остальных случаях - всё равно придется
доставать записи и сортировать их ручками. Поскольку проект всё таки тестовый, и базы данных мы
в нем не касаемся - решил хранить все записи просто [списком {словарей}] без дополнительных ключей.
