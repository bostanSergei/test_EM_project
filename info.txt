Привет! Во первых, спасибо за интересный проект. Это мое первое тестовое задание и оно получилось достаточно интересным!

Для работы в режиме администратора нужно будет ввести логин и пароль admin / admin

Администратор может изменять уже существующие записи. Это единственное отличие в функционале.
Пользователи, не прошедшие авторизацию - смогут создавать новые записи, искать юзеров по указанным паттернам
и пользоваться постраничным выводом всех юзеров из справочника.


Хранение данных - json - список словарей, где каждый элемент списка - словарь с ключами и значениями. Каждый словарь мы
может передать в класс user и получить доступ к атрибутам, свойствам и методам нашего базового класс и отдельного объекта

Все файлы названы в соответствии с той логикой, которую они выполняют:
baseLogic - стартовые функции программы. Запуск, авторизация, выбор действия, которое необходимо выполнить
changeUser - можем внести изменения в словарь конкретного пользователя
createNewUser - создание нового пользователя
findLogic - прописана функция поиска пользователей (в том числе и для внесения изменений)
info - тот файл, который вы сейчас читаете)

main - ФАЙЛ ЗАПУСКА. Здесь все модули собраны воедино. Именно этот файл мы запускаем для старта работы с нашим справочником

outByPage - постраничный вывод наших записей с предварительной сортировкой по имени или фамилии
user - класс пользователя
usersFile - json - файл, в котором хранятся наши записи
